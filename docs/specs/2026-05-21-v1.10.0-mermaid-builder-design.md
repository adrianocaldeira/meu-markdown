# v1.10.0 — Construtor Visual de Mermaid

**Data**: 2026-05-21
**Versão alvo**: v1.10.0 (MINOR)
**Tipo**: Feature

## Contexto

O Meu Markdown já renderiza blocos ```mermaid no preview (via `mermaid.min.js` em `Resources/`). A dor reportada pelo usuário é **lembrar a sintaxe** ao escrever os blocos — não a renderização. Esta feature adiciona dois caminhos complementares para facilitar a criação de diagramas Mermaid:

1. **Menu Inserir > Mermaid > Templates** — inserção rápida de esqueletos comentados, um por tipo de diagrama.
2. **Construtor visual** — janela WPF com formulário e preview ao vivo, para montar fluxogramas e diagramas de sequência sem escrever sintaxe.

## Decisões já tomadas

- **Abordagem técnica**: WPF nativo (não HTML/JS dentro de WebView2 como UI principal). Consistência com o resto do app, mesmo tema flat de `MessageDialog`/`InputDialog`, reuso de MVVM com `CommunityToolkit.Mvvm`.
- **Construtor é write-only no v1**: não parseia blocos Mermaid existentes para reconstruir modelo. Sempre insere bloco novo. Edição de bloco existente continua sendo edição de texto direto.
- **Construtor cobre 2 tipos no v1**: Flowchart e Sequence. Demais tipos têm apenas template no menu. Arquitetura preparada para adicionar tipos depois sem refazer base.
- **Preview ao vivo**: WebView2 dedicada dentro do construtor (não reusa o `MarkdownPreview` completo, que carrega Markdig, KaTeX, CSS GitHub-flavored).
- **Templates como recursos**: arquivos `.txt` embedded em `Resources/MermaidTemplates/`, não strings hardcoded em C#.

### Integração com app existente

- `MainWindow.xaml`: novo menu "Inserir" (ou submenu se já houver) com:
  - Itens `Mermaid > Fluxograma (template)`, `Mermaid > Sequência (template)`, … (8 itens, um por tipo).
  - Separador.
  - `Mermaid > Construtor visual…` que abre `MermaidBuilderWindow`.
- `MainViewModel`: comandos `InsertMermaidTemplateCommand(MermaidDiagramType)` e `OpenMermaidBuilderCommand`.
- `DocumentTabViewModel.InsertMermaidBlock(string mermaidCode)`: novo método que insere bloco ```mermaid…``` na posição do cursor do AvalonEdit ativo. Garante `\n\n` antes (se linha atual não-vazia) e `\n\n` depois. Posiciona caret depois do bloco.
- Inserção de template usa o mesmo `InsertMermaidBlock` (template já é o conteúdo do bloco).

### Fluxo de dados

```
Usuário escolhe item no menu
  │
  ├─ Template          → MainViewModel.InsertMermaidTemplateCommand
  │                       → MermaidTemplateService.GetTemplate(type)
  │                       → DocumentTabViewModel.InsertMermaidBlock(text)
  │
  └─ Construtor visual → MainViewModel.OpenMermaidBuilderCommand
                          → abre MermaidBuilderWindow (Owner=MainWindow, não-modal)
                          → ViewModel mantém modelo ativo (Flowchart ou Sequence)
                          → mudanças disparam debounce 300ms → MermaidPreviewLite.RenderAsync
                          → ao OK: ViewModel.ToMermaid() → janela retorna string → host insere bloco
```

## Especificação: Menu Inserir > Mermaid (templates)

### Itens

Os 8 templates ficam em `Resources/MermaidTemplates/` e são carregados via `Application.GetResourceStream`. Cada arquivo:

- Começa com a keyword Mermaid correta (`graph TD`, `sequenceDiagram`, `classDiagram`, `erDiagram`, `stateDiagram-v2`, `gantt`, `mindmap`, `pie`).
- Contém ao menos um comentário `%%` em PT-BR apontando onde editar.
- Está em UTF-8 sem BOM.

**Conteúdo de referência** (cada arquivo tem o esqueleto mínimo funcional + 1-2 elementos exemplares):

- `flowchart-td.txt` — `graph TD` com 3 nós (`A[Início]`, `B[Processo]`, `C[Fim]`) e 2 setas.
- `sequence.txt` — 2 atores (`Alice`, `Bob`) com 2 mensagens (envio + resposta).
- `class.txt` — 1 classe (`Animal`) com 2 campos e 1 método; 1 classe derivada com relação `<|--`.
- `er.txt` — 2 entidades (`CLIENTE`, `PEDIDO`) com relação `||--o{`.
- `state.txt` — `[*]` → estado A → estado B → `[*]`.
- `gantt.txt` — `dateFormat YYYY-MM-DD`, 1 section, 2 tarefas.
- `mindmap.txt` — raiz com 3 filhos e 1 neto.
- `pie.txt` — `pie title …` com 3 fatias.

### Inserção

- Se há seleção no editor, substitui a seleção pelo bloco.
- Se cursor está em linha não-vazia, prefixa `\n\n`.
- Sempre sufixa `\n\n`.
- Bloco delimitado por ` ```mermaid ` + newline + conteúdo + newline + ` ``` `.
- Caret final posicionado **após** o bloco inserido (consistente com o construtor; mantém previsibilidade).

### Atalhos de teclado

Nenhum no v1. Acesso só pelo menu.

## Especificação: Construtor visual

### Janela `MermaidBuilderWindow`

- **Modal**: não. Owner = MainWindow. Não bloqueia o app — usuário pode consultar o documento enquanto monta.
- **Tamanho**: inicial 1000×650, redimensionável. Lembra última posição e tamanho via mesmo mecanismo de outras janelas do app (se existir; senão, ficar em padrão fixo).
- **Tema**: flat consistente com `MessageDialog`/`InputDialog` (v1.9.0).
- **Fechamento**: botão "X" da janela equivale a Cancelar (com confirmação se houve edição).

### Layout

```
┌──────────────────────────────────────────────────────────┐
│ Tipo: [Fluxograma ▼]                                     │  ← topo: ComboBox
├────────────────────────┬─────────────────────────────────┤
│                        │                                 │
│  [Painel do tipo]      │   [MermaidPreviewLite]          │
│  ContentControl que    │   WebView2 com render Mermaid   │
│  carrega Flowchart-    │   atualizado debounce 300ms     │
│  Panel ou SequencePanel│                                 │
│                        │                                 │
├────────────────────────┴─────────────────────────────────┤
│ ▶ Ver código Mermaid (expander, read-only)               │
├──────────────────────────────────────────────────────────┤
│ ⚠ <mensagem de erro/aviso, se houver>                    │
├──────────────────────────────────────────────────────────┤
│                                  [Cancelar]  [OK]        │
└──────────────────────────────────────────────────────────┘
```

GridSplitter vertical entre painel esquerdo e preview, ajustável pelo usuário.

### ComboBox de tipo

Valores v1: `Fluxograma`, `Sequência`. Trocar com edição em andamento dispara `MessageDialog`: "Trocar de tipo descarta o que você fez. Continuar?". Estado vazio (modelo default) não dispara confirmação.

### Painel Flowchart

- **Direção**: RadioGroup horizontal: `TD` (default), `LR`, `BT`, `RL`.
- **Nós** (GroupBox "Nós"):
  - ListBox de `FlowNode`. ItemTemplate mostra `Id` + `Label` resumidos.
  - Botões: `+ Adicionar`, `Remover`, `↑`, `↓`. Mover desabilitado quando lista vazia ou item nas pontas.
  - Painel de edição do item selecionado (abaixo da ListBox):
    - `Id` (TextBox, auto-sugerido `n1`, `n2`, … no Add — usuário pode mudar)
    - `Label` (TextBox)
    - `Shape` (ComboBox: Retângulo `[…]`, Arredondado `(…)`, Losango `{…}`, Círculo `((…))`, Stadium `([…])`)
- **Arestas** (GroupBox "Arestas"):
  - ListBox de `FlowEdge`. ItemTemplate: `<FromId> → <ToId> (<Label>)`.
  - Botões: `+ Adicionar`, `Remover`.
  - Painel de edição:
    - `FromId` (ComboBox populado com IDs dos nós)
    - `ToId` (ComboBox idem)
    - `Arrow` (ComboBox: Sólida `-->`, Sólida c/ label `-->|…|`, Pontilhada `-.->`, Grossa `==>`)
    - `Label` (TextBox, habilitado só se Arrow=Sólida c/ label)

### Painel Sequence

- **Atores** (GroupBox "Atores"):
  - ListBox de `Actor`. Botões `+`, `−`, `↑`, `↓`.
  - Edição: `Name`, `Alias` (opcional), checkbox `É ator` (gera `actor`; desmarcado gera `participant`).
- **Mensagens** (GroupBox "Mensagens"):
  - ListBox de `Message`. ItemTemplate: `<From> → <To>: <Label>`.
  - Botões `+`, `−`.
  - Edição: `From` (ComboBox de atores), `To` (ComboBox), `Arrow` (síncrona `->>`, assíncrona `-)`, resposta `-->>` ), `Label`.

### Estado inicial

Ao abrir o construtor com tipo Flowchart, modelo vem com:
- Direção `TD`
- 2 nós: `n1[Início]`, `n2[Fim]`
- 1 aresta: `n1 --> n2`

Ao trocar para Sequence, modelo vem com:
- 2 atores: `Alice`, `Bob`
- 1 mensagem: `Alice ->> Bob: Olá`

Isso garante que o preview nunca aparece vazio.

### Expander "Ver código Mermaid"

Recolhido por default. Quando aberto: mostra `TextBox` read-only (multi-line, monospace) com o código gerado em tempo real. Botão "Copiar" ao lado.

### Botões

- **OK**: dispara `InsertMermaidBlock(model.ToMermaid())` no `DocumentTabViewModel` ativo da MainWindow, fecha janela. Se não houver documento ativo: desabilitado com tooltip "Abra um documento para inserir o diagrama".
- **Cancelar**: se modelo mudou em relação ao estado inicial → confirmação "Descartar diagrama?". Caso contrário, fecha direto.

### Validação leve

Validações **não bloqueiam** o OK; mostram aviso na barra de erro abaixo do expander:
- Aresta com `FromId` ou `ToId` apontando pra nó inexistente → "Aresta ignorada: nó '<id>' não existe."
- 0 nós (Flowchart) ou 0 atores (Sequence) → "Adicione pelo menos um elemento."
- ID de nó duplicado → "ID '<id>' duplicado. Apenas o primeiro será usado."

Ao gerar `ToMermaid()`, arestas órfãs são omitidas; IDs duplicados, ignorados (primeiro vence).

## Especificação: geração de Mermaid

### `FlowchartModel.ToMermaid()`

```
graph <Direction>
    <id1><ShapeOpen><LabelEscaped><ShapeClose>
    <id2><...>
    ...
    <fromId> <Arrow>[|<labelEscaped>|] <toId>
    ...
```

Mapeamento de Shape:
- Rectangle → `[label]`
- Rounded → `(label)`
- Diamond → `{label}`
- Circle → `((label))`
- Stadium → `([label])`

Mapeamento de Arrow:
- Solid → `-->`
- SolidLabeled → `-->|<label>|`
- Dotted → `-.->`
- Thick → `==>`

### `SequenceModel.ToMermaid()`

```
sequenceDiagram
    <actor|participant> <Name>[ as <Alias>]
    ...
    <FromName> <Arrow> <ToName>: <LabelEscaped>
    ...
```

Mapeamento de Arrow:
- Síncrona → `->>`
- Assíncrona → `-)`
- Resposta → `-->>`

### Escape / sanitização

Comum a ambos:
- Aspas duplas no label: trocadas por aspas simples (`'`).
- Quebras de linha (`\r\n`, `\n`) no label: trocadas por `<br/>` (Mermaid renderiza HTML em labels).
- Caracteres `[`, `]`, `(`, `)`, `{`, `}` em labels de Flowchart: envolver label inteiro entre aspas duplas dentro do shape (ex: `n1["Texto com (parênteses)"]`).
- ID vazio ou contendo espaço/caracteres não-alfanuméricos: gerar ID temporário `n_<hash8>` na hora de emitir; não corrompe modelo.

## Preview ao vivo

### `MermaidPreviewLite`

UserControl com `WebView2` ocupando 100% e método público `Task RenderAsync(string mermaidCode)`.

**Página HTML do preview** (carregada uma vez ao inicializar a WebView2; mantida como recurso embedded em `Resources/mermaid-preview-lite.html`):

- Apenas `<body>` com uma `<div id="diagram">`, um `<div id="error">` para mensagens, e referência a `mermaid.min.js`.
- Estilo mínimo: fundo da página alinhado ao tema, padding interno, fonte Segoe UI, mensagem de erro em vermelho.
- Inicializa Mermaid com `startOnLoad: false`.
- Expõe `window.render(code)` que:
  1. Limpa a `<div id="error">`.
  2. Chama `await mermaid.render('m', code)` para gerar o SVG.
  3. Em sucesso: substitui o conteúdo da `<div id="diagram">` pelo SVG gerado **pela própria biblioteca Mermaid** (origem confiável: o input não vem do documento aberto, vem do modelo do construtor que o próprio app produz).
  4. Em erro: limpa o diagrama, escreve `err.message` na `<div id="error">` (via `textContent`, não interpola HTML) e posta `{ type: 'error', message }` para o host via `chrome.webview.postMessage`.

**Resolução de `mermaid.min.js`**: via URI virtual configurada pela WebView2 (`SetVirtualHostNameToFolderMapping`) apontando para `Resources/` — mesmo padrão já usado pelo `MarkdownPreview` (se padrão atual for diferente, alinhar com o existente em vez de divergir).

### Debounce

`MermaidBuilderViewModel` mantém um `DispatcherTimer` de 300ms. Hooks no ViewModel:
- Trocar modelo (mudar tipo) → cancela timer e renderiza imediatamente.
- Toda mudança em `ObservableCollection` (CollectionChanged) e em propriedades dos itens (PropertyChanged propagado) → restart do timer.
- Tick do timer → `RenderAsync(model.ToMermaid())`.

Helper interno conecta/desconecta handlers de `PropertyChanged` ao adicionar/remover itens da coleção (evita leaks).

### Tratamento de erro

Host recebe `{ type: 'error', message }` via `CoreWebView2.WebMessageReceived`. ViewModel expõe `ErrorMessage` (string?) bindado a um TextBlock vermelho abaixo do expander.

### Tema

v1: tema `default` do Mermaid (fundo branco). Se o app já tem detecção de tema escuro/claro, passar para a página antes do primeiro render e usar `mermaid.initialize({ theme: 'dark' | 'default' })`. Caso contrário, deixar `default` e tratar em iteração futura.
